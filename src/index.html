Multiple GitHub accounts on a single PC

Jun 24, 2021

Issue

  • Windows saves the GitHub credentials to Credential Manager. You have to remove it every time you want to push or pull as another GitHub account.
  • For example, you want to push and pull as a work account and personal account in the work PC.

Overview

  • Make SSH keys for each work and personal accounts.
  • Add the SSH keys to each GitHub work and personal account settings.
  • In ~/.gitconfig file, update git global configuration.
  • In ~/.ssh/config file, update SSH configuration.
  • If you already cloned a repository before the above steps, update the remote URL.

If you already cloned a repo before this

  • Work repository
    • $ git remote set-url REMOTE_NAME git@github.com-work:WORK_NAME/REPOSITORY_NAME.git
      • REMOTE_NAME is usually origin
      • WORK_NAME should be your company organization name in GitHub
  • Personal repository
    • $ git remote set-url REMOTE_NAME git@github.com-work:USERNAME/REPOSITORY_NAME.git
      • REMOTE_NAME is usually origin
      • USERNAME should be your personal GitHub account name.

If you clone a repo from now

  • git clone git@github.com-work:WORK_NAME/REPOSITORY_NAME.git
    • WORK_NAME should be your company organization name in GitHub
  • git clone git@github.com-personal:USERNAME/REPOSITORY_NAME.git
