Root identity and administrator

Jun 30, 2021

Root identity (root account credential)

  • The combinarion of the email address and password which is used to create an AWS account initially.
  • The root has all the permissions. Too many permissions for everyday use.
  • AWS recommends not to use the root account for everyday use. With the administrator and account alias described below, sign in to AWS resources becomes easier.

Administrator

  • The conventional name for the everyday use accounts.
  • There is no such account by default, so we create a new user Administrator and create a new group Administrators, and make the user belong to the group.
  • Go to IAM, Users, Make a user, Administrator
  • Groups, Make a group, Administrators
    • This group has an AdministratorAccess policy

Account alias

  • By default, all the AWS accounts are automatically assigned a unique 12 digit numbers, but it's hard to remember it, so we give a name.
  • Not only to hide Account ID from the console, sign-in becomes easier by typing this meaningful alias rather than random 12 digit numbers.
  • Go to IAM, Dashboard, click Customize next to Sign-in URL, type the account alias.

Sign in to AWS

  • Use account alias, administrator name, administrator password are to sign in AWS.
  • We can avoid security question which appears all the time when you try to sign in by the root account.

Reset password of a user

  • Go to Security credentials tab of the user.
  • Click manage next to Console password.
  • Set password, Custom password, type a new password.

Resource

  • Create your first IAM admin user and user group
  • Reset user password
